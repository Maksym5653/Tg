# NickCatcher — хакерський трекер telegram-ніків

Android-застосунок з плаваючою кнопкою поверх усіх додатків, який оцінює
приблизну вартість telegram-нікнейму та веде спільний рахунок гравців.

## Що вже зроблено
- Плаваюча кнопка `$` (OverlayService) — рухається, тягнеться, працює поверх будь-якого додатку.
- Клік по кнопці відкриває міні-вікно: вводиш `@nickname` → тисни "ОЦІНИТИ" →
  бачиш приблизну ціну. Повторний клік по кнопці `$` закриває вікно.
- Вхід за іменем при першому запуску. Якщо ім'я — **"Максим"** (без регістру),
  користувач отримує права адміна.
- Головний екран показує список усіх гравців, їхні зловлені ніки з цінами,
  Earned на людину, "Everyone earned today" та "Earned for all time" —
  дані спільні для всіх через Firebase Realtime Database, оновлюються в реальному часі.
- Адмін (Максим) бачить кнопку `[x]` біля кожного ніку і може видаляти будь-який запис.
- Хакерська стилістика: чорний фон, матричний зелений моноширинний текст.

## ⚠️ Важливі чесні обмеження

1. **Пошук тепер реально йде через Gemini 2.5 Flash з Google Search grounding**
   (`NicknameValuator.kt` → `askGemini()`). Модель сама шукає в інтернеті
   (аукціони Fragment, форуми перепродажу) і повертає число-ціну. Якщо запит
   до Gemini не спрацював (немає мережі, невалідний ключ, вичерпана квота) —
   автоматичний відкат на локальну евристику за довжиною/патернами ніку, щоб
   кнопка ніколи просто не "мовчала". У попапі видно, звідки прийшла ціна:
   `(gemini)` чи `(locally)`.

2. **Про ключ, який ти дав** (`AQ.Ab8RN6LGmHMHhc-r_E7dN1OLhyt2yTmMCVocqN1rJPoO-aaCiw`),
   лежить у `Config.kt`. Формат цього ключа відрізняється від типових ключів
   Google AI Studio (ті зазвичай починаються з `AIzaSy...`). Схоже на
   тимчасовий/сесійний OAuth-токен Google, а не постійний API-ключ Generative
   Language API. У мене немає мережі, щоб перевірити його наживо. Якщо після
   збірки побачиш, що ціна завжди приходить `(locally)`, а не `(gemini)` —
   це і буде ознакою, що ключ не підходить. Тоді:
   - зайди на https://aistudio.google.com/apikey,
   - створи новий ключ ("Create API key"),
   - встав його в `Config.kt` замість поточного.

3. **Ключ у коді застосунку видно кожному, хто розпакує APK** (apktool/jadx —
   2 хвилини роботи). Для приватного проєкту "для своїх" це прийнятний ризик,
   але не викладай APK у публічний доступ із живим платним ключем. Якщо
   захочеш закрити цю дірку — виклик Gemini можна перенести на Firebase Cloud
   Function і тримати ключ лише на сервері.

4. **Я не можу скомпілювати готовий APK у цьому чаті** — тут немає Android SDK
   і інтернету для завантаження Gradle-залежностей. Нижче — покрокова інструкція,
   як зібрати APK самостійно за ~10 хвилин.

## Як зібрати APK

### 1. Встанови Android Studio
https://developer.android.com/studio

### 2. Відкрий проєкт
`File → Open` → вибери папку `NickCatcher` (цю, з файлом `settings.gradle`).

### 3. Підключи Firebase (потрібно для спільних даних усіх гравців)
1. Зайди на https://console.firebase.google.com → "Add project" → назви як хочеш.
2. У проєкті натисни "Add app" → Android → пакет `ua.hackteam.nickcatcher`.
3. Завантаж файл `google-services.json` і поклади його в папку `NickCatcher/app/`
   (поруч із `build.gradle` модуля app).
4. У лівому меню Firebase зайди в **Build → Realtime Database → Create Database**.
   Обери режим **test mode** для старту (потім можна налаштувати правила безпеки
   нижче).
5. Рекомендовані правила (Realtime Database → Rules), щоб усі учасники читали/писали:
   ```json
   {
     "rules": {
       ".read": true,
       ".write": true
     }
   }
   ```
   Це відкриті правила для приватного невеликого проєкту серед друзів. Для
   продакшн-використання варто додати аутентифікацію.

### 4. Зібрати APK
`Build → Build Bundle(s)/APK(s) → Build APK(s)`.
Готовий файл буде в `app/build/outputs/apk/debug/app-debug.apk`.

### 5. Встановити на телефон
Перекинь `app-debug.apk` на телефон і встанови (дозволь встановлення з
невідомих джерел). При першому запуску:
- введи своє ім'я (якщо ти головний — введи рівно "Максим");
- дозволь "Показ поверх інших застосунків", коли попросить система;
- з'явиться зелена кнопка `$` — тягай її куди зручно, тисни, вводь ніки.

## Автоматична збірка через GitHub Actions

У проєкті є `.github/workflows/android-build.yml` — при кожному push у
`main`/`master` (або вручну кнопкою "Run workflow" у вкладці Actions)
GitHub сам збере `app-debug.apk` і покладе його в артефакти запуску
(вкладка **Actions → відповідний запуск → Artifacts**).

### Секрети, які потрібно додати в репозиторій
`Settings → Secrets and variables → Actions → New repository secret`:

| Назва секрету            | Що покласти                                                                 |
|---------------------------|------------------------------------------------------------------------------|
| `GOOGLE_SERVICES_JSON_B64`| Вміст `google-services.json` у base64: `base64 -w0 google-services.json` (Linux) або `base64 -i google-services.json` (macOS) |
| `GEMINI_API_KEY`          | Твій ключ Gemini (той самий, що зараз лежить у `Config.kt`)                 |

Без цих секретів workflow все одно збере APK, просто Firebase і Gemini не
запрацюють, поки ти їх не додаси.

`.gitignore` вже налаштований так, щоб `google-services.json` і зібрані
`*.apk` ніколи не потрапили в git — ключі підставляються тільки під час
збірки в Actions.
